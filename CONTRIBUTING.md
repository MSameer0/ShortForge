# Contributing to ShortForge

First off, thank you for considering contributing to ShortForge! It's people like you that make this tool great.

## How Can I Contribute?

### Reporting Bugs
If you find a bug, please check if it has already been reported. If not, open a new issue and include:
* A clear and descriptive title
* Steps to reproduce the bug
* Expected vs actual behavior
* Your OS and Python version

### Suggesting Enhancements
Feature requests are always welcome. Please open an issue to propose a new feature. Be sure to explain:
* What the feature is
* Why it would be useful
* Any potential alternatives you've considered

### Submitting Pull Requests
1. Fork the repository and create your branch from `main`.
2. Ensure your development environment is set up by installing the requirements:
   ```bash
   pip install -r requirements.txt
   ```
3. Make your changes.
4. Format your code. We use `black` for code formatting to keep the codebase consistent:
   ```bash
   black .
   ```
5. Test your changes to ensure nothing is broken.
6. Submit a pull request! Please provide a detailed description of what you changed and why.

## Code Style Guide
* We use [Black](https://github.com/psf/black) as our code formatter.
* Keep comments meaningful. We prefer code that documents itself through clear variable and function names over excessive inline comments.
* Stick to the existing dark theme aesthetic for any UI contributions.

Thank you for contributing!
